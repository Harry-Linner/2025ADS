# MapReduce 词频统计测试结果说明

## 测试环境
- **系统**: Linux (WSL)
- **编译器**: GCC C99标准
- **系统信息**: `tests/system_info.txt`

## 测试数据集

### 1. 小规模测试
- **文件**: `tests/data/small_test.txt` (96个单词)
- **用途**: 功能验证

### 2. 中等规模测试  
- **文件**: `tests/data/medium_test.txt` (10,000个单词)
- **用途**: 中等规模性能分析

### 3. 大规模测试
- **文件**: `tests/data/large_test.txt` (120,000个单词)
- **用途**: 满足作业要求(100k+单词)，主要性能对比数据

### 4. 真实数据测试
- **文件**: `testcase/Frankenstein.txt` (~78,000个单词)
- **用途**: 真实世界数据验证

## 🎯 报告中使用的关键文件

### 📊 主要图表（必须包含）
- **`tests/performance/performance_comparison.png`** ⭐ **报告主图**
  - 串行vs并行执行时间对比
  - 加速比分析
  - 英文标签，无乱码

### 📈 主要数据源（必须引用）
- **`tests/performance/performance_report.txt`** ⭐ **主要数据源**
  - 平均执行时间
  - 标准差和可靠性指标
  - 加速比计算
  - 多次运行统计(每个测试5次)

### ✅ 算法正确性验证
- **位置**: `tests/results/`
- **验证结果**: 所有测试显示串行和并行算法结果完全一致
- **用途**: 证明算法正确性

```

中等规模 (10,000个单词):  
- 串行: ~0.03秒, 并行: ~0.04秒, 加速比: 0.74x

大规模 (120,000个单词):
- 串行: ~0.27秒, 并行: ~0.28秒, 加速比: ~1.0x

Frankenstein (78,000个单词):
- 串行: ~1.21秒, 并行: ~0.36秒, 加速比: 3.32x
```

## 📝 报告建议

### 报告中应包含：
1. **主要图表**: `performance_comparison.png`
2. **性能数据**: `performance_report.txt`中的统计数据
3. **测试规模**: 96, 10k, 120k, 78k个单词
4. **验证结果**: 所有算法产生相同结果

### 不要包含的文件：
- `*.log` (调试日志)
- `*_times.txt` (原始计时数据)  
- `*_run*.txt` (单次运行输出)
- 测试脚本本身
